Configuração do Windows

1. Habilitar WSL2

# Habilitar WSL dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart # Habilitar Virtualização dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart # Definir WSL2 como padrão wsl --set-default-version 2

2. Windows Terminal

TERMINAL FEATURES ├── Multi-tabs ├── Temas personalizados ├── Split panes └── Integração WSL

3. Variáveis de Ambiente

# Adicionar ao Path [Environment]::SetEnvironmentVariable( "Path", [Environment]::GetEnvironmentVariable("Path", "Machine") + ";C:\your\path", "Machine" )

Configuração do macOS

1. Homebrew

# Instalar Homebrew /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)" # Verificar instalação brew doctor

2. Command Line Tools

# Instalar Command Line Tools xcode-select --install # Verificar instalação xcode-select -p

3. Terminal Enhancement

# Instalar iTerm2 brew install --cask iterm2 # Instalar Oh My Zsh sh -c "$(curl -fsSL https://raw.github.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

Configuração do Linux

1. Pacotes Essenciais

# Ubuntu/Debian sudo apt update sudo apt install build-essential curl wget git # Fedora sudo dnf groupinstall "Development Tools" sudo dnf install curl wget git

2. Terminal Setup

# Instalar e configurar ZSH sudo apt install zsh chsh -s $(which zsh) # Instalar Oh My Zsh sh -c "$(curl -fsSL https://raw.github.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

Configurações Comuns

1. Git Global

# Configuração básica git config --global user.name "Seu Nome" git config --global user.email "seu@email.com" # Editor padrão git config --global core.editor "code --wait"

2. SSH Keys

# Gerar nova chave SSH ssh-keygen -t ed25519 -C "seu@email.com" # Iniciar ssh-agent eval "$(ssh-agent -s)" # Adicionar chave ssh-add ~/.ssh/id_ed25519

Ambientes de Desenvolvimento Isolados

SDKMAN! com Docker

FROM ubuntu:22.04 # Instalar dependências RUN apt-get update && apt-get install -y \ curl \ zip \ unzip # Instalar SDKMAN! RUN curl -s "https://get.sdkman.io" | bash ENV SDKMAN_DIR="/root/.sdkman" # Configurar ambiente COPY .sdkmanrc /app/.sdkmanrc WORKDIR /app # Instalar SDKs RUN bash -c "source $SDKMAN_DIR/bin/sdkman-init.sh && \ sdk env install"

Gerenciamento de Versões por Projeto

PROJECT STRUCTURE ├── .sdkmanrc # Versões do projeto ├── .java-version # Alternativa para Java └── docker-compose.yml

Exemplo de docker-compose.yml

version: '3.8' services: java-dev: build: . volumes: - .:/app environment: - SDKMAN_DIR=/root/.sdkman

Scripts de Automação

#!/bin/bash # setup-dev.sh # Verificar SDKMAN! if ! command -v sdk &> /dev/null; then echo "Instalando SDKMAN!..." curl -s "https://get.sdkman.io" | bash fi # Carregar SDKMAN! source "$HOME/.sdkman/bin/sdkman-init.sh" # Instalar versões do projeto if [ -f .sdkmanrc ]; then echo "Configurando ambiente do projeto..." sdk env install fi

Troubleshooting

Problemas Comuns

  1. WSL não inicia

    • Verificar virtualização no BIOS

    • Reinstalar WSL

    • Atualizar Windows

  2. Homebrew falha

    • Verificar permissões

    • Atualizar macOS

    • Reinstalar Xcode CLI

  3. Linux packages quebrados

    • Limpar cache

    • Atualizar repositórios

    • Verificar dependências
